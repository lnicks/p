User Expectations

Full content available on mobile

The 2016 comScore report says US users spend 60% of screen time in front of mobile devices.(footnote)

Easy navigation without distractions

On the average Web page, users have time to read at most 28% of the words during an average visit; 20% is more likely.

No Waiting

  • 49% of all mobile users expect load times <2 seconds. That number has been steadily increasing for over 5 years. This number increases outside of the country.
  • 30% of users expect <1%, meaning instantaneous, compared to 5% in 2009.

Source: Akamai, Performance mattters

So the users want...

  • 100% of content available on a desktop site...
  • without having to hunt for it...
  • and they want it now.

How do you go about making all of these things happen at once?

How is the content delivered?

Understand how the content is delivered (CMS, API) and limitations of the platform.

  • 45% of top 10 million websites use a known CMS (W3Techs, 2016)
  • Content structure / stored might have an impact on how it is presented

Draw it out

  • Site Maps
    • Establishes scope of the entire website
    • Hierarchy provides foundation for site navigation
    • Archived / Legacy content doesn't get left out
  • Flow Diagrams
    • Sitemaps provide a basis for navigation, flowcharts show the path a user takes
    • UI pathways don't always follow a traditional 'page' model

Design

Consider Context

  • Wireframes / style tiles help users visualize content placement & layout
  • Prioritize content placement on a mobile vs desktop
  • Responsive Flow & Progressive information disclosure

Responsive flow

  • Distill content into chunks
  • Design for individual elements to be placed into a larger page
  • Establish breakpoints and decide where content / elements will be placed

Navigation

The Physics behind a condensed screen size means that there will always conflict between the screen size and the amount of information you can put onto it. There comes a point where usability is lost by not providing enough navigation (NNGgroup)

Site map shouldn't change, just how the content is presented

NNGroups Tips for navigation

  • If your site has 4 or fewer top-level navigation links, display them as visible links
  • If your site has more than 4 top-level navigation links, the only reasonable solution is to hide some of these
  • In page structures to link to related information

Does device and size matter?

  • A year ago, there were over 24,000 different web enabled android devices (IMG)
  • Screensize & aspect ratio are just as inconsistent (IMG)
  • Design for common breakpoints / grid solutions

Flat Designs

Pros:

  • Easy to read, typography & color focused
  • Removes UX / Code complexity: performance, grid / framework friendly!
  • Streamlines content & flow through chunking
  • o Flexible, reusable, content independent layout

Flat Designs

Cons:

  • Can be over flattened, impacting UX
  • Sensitive to colors (branding & accessibility issue)

Boostrap Primer

Which front end framework to use

We need to be flexible enough to use any of them, as we don’t always have the luxury to start the project from scratch

Bootstrap is the most popular, with version 4 being worked on soon

Demo template for this workshop at : XXX

New in Bootstrap 4

In v4, Bootstrap came up with it’s unique browser defaults unification code named Reboot. Reboot mostly follows Normalize.css adding two main concepts of its own

  • Typographical measures are all in rems
  • Only one sided vertical margin is being used, margin-bottom. As vertical margins can collapse, yielding unexpected results

HTML Behaviors in Bootstrap

  • The box-sizing attribute is globally set on every element—including *:before and *:after , to border-box . This ensures that the declared width of element is never exceeded due to padding or border.
  • A base font-size: 16px is declared on the <html> and font-size: 1rem on the <body> for easy responsive type-scaling via media queries.
    • <

Further HTML Behaviors in Bootstrap

  • The <body> also sets a global font-family and line-height; this is inherited later by some form elements to prevent font inconsistencies
  • For safety, the <body> has a declared background-color, defaulting to #fff

Further HTML sBehaviors in Bootstrap

  • <h1> - <h6>, <p> tags have margin-top removed. <h1> - <h6> have margin-bottom:0.5rem and <p> has margin-bottom: 1rem
  • <ul>, <ol> and <dl> have margin-top removed and margin-bottom: 1rem

Typography and Utility Classes

  • Use a native font stack that selects the best font-family for each OS and device
  • It is recommended to use the $font-family-base, $font-size-base, and $line-height-base attributes for <body>tag
  • Global link color via $link-color and underlines need to be applied only on :hover
  • Use $body-bg to set a background-color on the <body> (#fff by default)
  • .lead component class has been added to make the paragraph stand out
  • If you want to make the paragraphs styled as <mark> and <small> you should use .mark and .small utility classes

Bootstrap utilities are basically classes with single purpose, they help as minimize the amount of CSS we write.

Naming format is the following: {property}-{sides}-{size}

  • m stands for margin
  • p stands for padding

Sides

  • t - for classes that set margin-top or padding-top
  • b - for classes that set margin-bottom or padding-bottom
  • l - for classes that set margin-left or padding-left
  • r - for classes that set margin-right or padding-right
  • x - for classes that set both *-left and *-right
  • y - for classes that set both *-top and *-bottom
  • a - for classes that set a margin or padding on all 4 sides of the element

Sizes

  • 0 - for classes which eliminate the margin or padding by
    setting it to 0
  • 1 - (by default) for classes that set the margin or padding to $spacer-x or $spacer-y
  • 2 - (by default) for classes that set the margin or padding to $spacer-x * 1.5 or $spacer-y * 1.5
  • 3 - (by default) for classes that set the margin or padding to $spacer-x * 3 or $spacer-y * 3

Bootstrap 4 cards

Cards are probably the most exciting new feature in Bootstrap 4

The card is basically a Flexible and Extensible Container of all types of content, including Headers, Footers, HTML tags, images, etc…

In Bootstrap 4 cards replace old panels, wells and thumbnail of Bootstrap 3
