What does 'Accessible' mean?

“the degree to which a product, device, service, or environment is available to as many people as possible”

source:wikipedia

Why is accessibility important?

  • Adaptive software development hasn’t been part of mainstream, so new tech often causes new problems, excluding groups of people almost at random.
  • Design needs to be considered early in projects to ensure access for all.
  • Include testing by people with disabilities during a project.

Legal Risks

Accessible alternatives must provide equivalent functionality. So if a website is inaccessible, any alternative must be available 24/7, from any location, etc.

  • For educational institutions, the Deptartment of Justice has recently held that Section 504 (from 1973) requires all programs and services (including digital) to be equally accessible to all.
  • The end result for the student is what natters, not using specific techniques.

Guidelines and Policies

WCAG 2.0 (Level A, AA, AAA)

  • Increasing levels of specificity (& difficulty)
  • Level AA compliance (which includes Level a) has gained acceptance as international standard

Section 508

  • Regulations for Federal websites and contractors from 2001;
  • Scheduled for "Refresh" in 2016
  • Meeting WCAG2 Level AA considered as meeting the new 508 standard.

Electronic Accessibility Leadership Team

  • UC committee of campus representatives and other stakeholders
  • Developing systemwide accessibility initiatives for software, training programs, procurement guidelines.

User Scenarios

  • Learning Disabilities / Cognitive Impairment
  • Deaf / Hearing Impairment
  • Dexterity / Physical Impairment
  • Low Vision, color blindness
  • Blind

Learning Disabilities / Cognitive Impairment

  • Largest category, but least understood (tech solutions still evolving)
  • Avoid strobing effects (can cause seizures)
  • Allow customization for a low-distraction interface, where appropriate
  • Many of these features will be useful to any user in an unpredictable environment (noise, glare, etc.)

Deaf / Hearing Impairment

  • Can use visual interface
  • Need visual cues for sound events in apps
  • Need captioning for multimedia content
  • Plan apps to display captioned video
  • Provide mechanism for uploading of captioned video and caption files

Dexterity / Physical Impairment

  • May have difficulty with small buttons, interacting with moving elements, etc.
  • Provide large target areas for tap events
  • Offer interface customization options if applicable
  • Visual focus indicators are critical to users who rely on keyboard accessibility

Low Vision, color blindness

  • Likely to use magnification (Zoom in iOS, etc.)
  • Provide color contrast of 4.5:1 for standard text
  • Avoid problem color combinations
  • Avoid text embedded in graphics, which will not let users resize

Blind

  • Will have the most difficulty with standard visual interface
  • Primary users of iOS VoiceOver utility

State of Mobile Accessibility

  • Goal of web apps is to be cross-platform/interoperable
  • Accessibility Tools do most of the work but are specific to mobile operating system
  • Wide gap in standards, old standards are not up to date with advancing mobile technologies
  • Best strategy for web developers: Use MWBP, with understanding of WCAG 2.0 and awareness of built-in accessibility features of major target platforms
  • Tools like Phonegap/Cordova can save much pain involved in developing for multiple platforms

Voice Over

  • Apple VoiceOver utility
  • Built-in text-to-speech
  • Rapid response to user actions
  • Two methods of navigation: Flick left/right or tap active screen areas
  • Altered set of gestures for more reliable interaction

Android TalkBack

  • Built-in text-to-speech
  • Applies to devices that run Android, not just smartphones / tablets
  • More User Customizations available (gesture to command)
  • Heavy customizations, OS fragmentation, & device variability decrease consistency

Native Platform Tips

iOS

  • Many disability-friendly features and utilities
  • Easiest platform for developers to provide accessibility (UIKit Accessibility API)

Android

  • Many accessibility utilities, but requires more configuration by user
  • More difficult dev environment: must enable Accessibility API

Web & Cross Platform Tips

PhoneGap/Cordova:

  • Has Accessibility API
  • Best choice for multi-platform development

Web

  • Can leverage existing HTML/responsive design skill sets
  • Requires work to meet Web Content Accessibility Guidelines (WCAG 2.0, Level AA)

Manual Accessibility Testing

Quick Tests

  • Automated tests still show many items that require human judgment of accessibility (AMP scores show either 100% or 0% success)
  • Practical / Case based analysis
  • Tests for elements that can only be evaluated by a human
  • Non technical assessment tools

Quick Test Scope

  • We aim to create a fast process to make many of those human-judgment assessments
  • We do not attempt to show how to fix accessibility errors in this talk

Quick Tests Goals

  • Further prioritize the 21 Level A WCAG checkpoints
  • Do verifiable, repeatable tests for the 13 AA items (that mostly require human judgment)
  • Find simple, quick tests that can be applied to large amounts of content

The Tests

  • Consistent Navigation
  • Color Contrast
  • Double-Size Test
  • Copy Test
  • Logical Structure
  • Focus Visible
  • Captions
  • Image Accessibility

Consistent Navigation

3.2.3 Consistent Navigation: Navigational mechanisms that are repeated on multiple Web pages within a set of Web pages occur in the same relative order each time they are repeated, unless a change is initiated by the user (Level AA)

3.2.4 Consistent Identification: Components that have the same functionality within a set of Web pages are identified consistently (Level AA)

Consistent Navigation

Maximum 5 min: Are there unusual changes to the navigation or other page components?

Things to Check:

  • Consistency between Front page and secondary page navigation
  • Choose a common navigation element and use keyboard only to get there

Color Contrast

1.4.3 Contrast (Minimum): The visual presentation of text and images of text has a contrast ratio of at least 4.5:1 (Level AA)

Color Contrast

Run tests on 4-5 potential problem areas, concentrating on extreme cases requiring immediate fix

  • WebAIM’s Jared Smith mentions "readability is a continuum", as you can always be more accessible
  • Text at 4.4:1 isn’t suddenly totally illegible. So focus on the biggest problems

Double Size Test

1.4.4 Resize text: Except for captions and images of text, text can be resized without assistive technology up to 200 percent without loss of content or functionality. (Level AA)

  • Use CTRL-+ (CMD-+) to test whether the site is readable at 2x magnification

Copy Test

1.4.5 Images of Text: If the technologies being used can achieve the visual presentation, text is used to convey information rather than images of text except for the following: (Level AA)

  • Text that cannot be copied is likely to be inaccessible
  • To test if images of text are being used, try copying any “suspicious” items

Logical Structure

2.4.6 Headings and Labels: Headings and labels describe topic or purpose. (Level AA)

  • Headings should not be used for visual effects, on large blocks of content
  • Use judiciously (too much structure without content is a problem)

Logical Structure

  • Are headings used for logical structure?
  • Does structure make sense?
  • Do not worry about skipping consecutive heading levels

Visible Focus

2.4.7 Focus Visible: Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible. (Level AA)

  • Is there a clear focus indicator when tabbing with keyboard
  • On the homepage & 5 subpages, test focus by tabbing through the page

Captions

1.2.2 Captions (Prerecorded): Captions are provided for all prerecorded audio content in synchronized media, except when the media is a media alternative for text and is clearly labeled as such. (Level A)

Captions

  • Quantify & assess portion of content that is multimedia
  • Ensure controls are easy to identify and to enable captioning
  • If significant in quantity or importance, play 30 sec of 3-5 clips
    • Are captions present?
    • Are they readable?

Images

  • Survey Content: Appropriate Use of ALT text on Images
  • Ensure controls are easy to identify and to enable captioning:
    • Fail if several of those images have alt text = the filename
    • Fail if lots of text is embedded in images
